What's Inside Claude Code's Brain?

// the system prompt — your AI's operating manual

SYSTEM PROMPT — EXCERPT
"The user will primarily request you to perform software engineering tasks. These may include solving bugs, adding new functionality, refactoring code, explaining code, and more. When given an unclear or generic instruction, consider it in the context of these software engineering tasks."
🔧

It assumes you're a developer.

Claude Code's very first instruction is about software engineering. If you're a copywriter, designer, or PM — it might over-engineer simple tasks. This isn't a bug. It's baked into its DNA.

SYSTEM PROMPT — EXCERPT
"Carefully consider the reversibility and blast radius of actions. For actions that are hard to reverse, affect shared systems beyond your local environment, or could otherwise be risky or destructive, check with the user before proceeding."
🛡️

It's told to ask permission.

When Claude Code asks "Are you sure?" — it's not being annoying. It's following explicit instructions to protect you from irreversible actions. The phrase "blast radius" is literally in the prompt.

SYSTEM PROMPT — EXCERPT
"Avoid over-engineering. Only make changes that are directly requested or clearly necessary. Keep solutions simple and focused. Don't add features, refactor code, or make 'improvements' beyond what was asked."
✂️

Even it is told: don't over-engineer.

Claude Code is explicitly instructed to stay focused and minimal. But its definition of "simple" is still through an engineer's lens — a simple shell script might not feel simple to you.

SYSTEM PROMPT — EXCERPT
"Go straight to the point. Try the simplest approach first without going in circles. Be extra concise. Lead with the answer or action, not the reasoning. Skip filler words, preamble, and unnecessary transitions."
💬

That's why it sounds so terse.

Compared to claude.ai, Claude Code feels blunt. That's not a personality — it's a style instruction. It's told to skip the warmth and get to work. Great for code. Jarring for brainstorming.
